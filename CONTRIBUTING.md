# Contributing

Thanks for your interest in contributing to this project.

## Ways to Contribute

- Report bugs and request features through GitHub issues
- Improve documentation
- Submit fixes and enhancements via pull requests

## Development Setup

1. Fork and clone the repository.
2. Install dependencies:

   ```bash
   npm install
   ```

3. Build the action bundles:

   ```bash
   npm run build
   ```

## Pull Request Guidelines

- Keep pull requests focused and small when possible.
- Include a clear description of the change and why it is needed.
- Update documentation when behavior or inputs/outputs change.
- Rebuild `lib/` artifacts when source files under `src/` are modified.

## Commit Messages

Use clear, descriptive commit messages.

## Reporting Security Issues

Please do not report security vulnerabilities in public issues. See [SECURITY.md](./SECURITY.md) for private reporting instructions.

## Code of Conduct

By participating in this project, you agree to follow the [Code of Conduct](./CODE_OF_CONDUCT.md).
